1. Installer phpBB3

Pour l’installation, la mise à jour et la conversion, les instructions à suivre sont indiquées dans le document INSTALL [ndt :INSTALL (vo)] fourni avec cette distribution. Si vous avez l’intention de convertir votre forum depuis une version phpBB 2.0.x, nous vous recommendons fortement d’effectuer une sauvegarde de vos données avant de vous lancer dans la procédure !

Les utilisateurs de l’une des versions Béta de phpBB3 ne peuvent pas effectuer une mise à jour directement vers phpBB3.

Merci de noter que nous n’apporterons pas d’assistance pour les installations suivantes :

  • Mises à jour depuis une version Béta de phpBB3 vers une version de phpBB3 RC1, ou supérieure ;
  • Conversions depuis une version phpBB 2.0.x vers toute version Béta de phpBB3 ;
  • Installations d’une version Béta de phpBB3.

Bien sûr, nous apporterons de l’assistance aux installations suivantes :

  • Mises à jour depuis une version phpBB3 RC1 vers la dernière version de phpBB3 ;
  • Conversions depuis une version phpBB 2.0.x vers la dernière version de phpBB3 ;
  • Nouvelles installations de phpBB3 - uniquement vers la dernière version de phpBB3.

2. Utiliser phpBB3

Une fois installé, vous pouvez facilement gérer phpBB3 grâce aux panneaux d’administration ou modération. Si vous avez besoin d’aide ou de conseil sur phpBB, consultez la Section 3 ci-dessous.

2.i. L'internationalisation (i18n)

De nombreux packs de langue et de nombreux packs de style sont disponibles. Vous pouvez les trouver sur la page officielle des téléchargements :

http://www.phpbb.com/downloads/

Il s’agit de la page officielle pour toutes les langues supportées. Si vous téléchargez un pack depuis un autre site tiers, vous le faites en comprenant que nous ne serons pas en mesure de vous aider. Aussi, ne demandez pas notre assistance si tel est votre cas !

L’installation des ces packs est simplissime : téléchargez le pack désiré, et décompressez ses fichiers dans le répertoire languages/. Faites attention à respecter l’arborescence des répertoires en faisant cela ! Une fois cette étape réalisée, allez dans la Panneau d’administration onglet Système, menu packs de langue (notation : Admin->Système->Packs de langue), et installez le pack de langue nouvellement listé. Pour installer le pack d’images d'un style, vous devez télécharger le pack correspondant au langage, et désarchiver le ou les fichiers dans le dossier correspondant à votre pack d'images de style styles/prosilver/imageset ou styles/subsilver2/imageset. Encore une fois, vous devez conserver la structure de l'arborescence des répertoires. Une fois installé, le pack d'images est immédiatement disponible.

Si votre langue n'est pas disponible, visitez nos forums, et vous trouverez une discussion sur la liste des traductions, actuellement disponibles ou en préparation. Cette discussion vous informera également au cas où vous désireriez devenir bénévole pour traduire en une langue non répertoriée.

2.ii. Les styles

Bien que le groupe phpBB soit plutôt fiers des styles déjà inclus, nous réalisons qu'ils peuvent ne pas être au goût de tout le monde. Aussi phpBB3 permet de changer de style avec une relative facilité. Tout d'abord, vous avez besoin de trouver et de télécharger le style que vous souhaitez. Nous maintenons un site à l'url :

http://www.phpbb.com/styles/
voire le lien de démonstration des styles de phpBB-fr.com.

Attention : les styles de tiers, élaborés pour phpBB2, ne seront pas comptibles avec phpBB3.

Une fois téléchargé l'archive du pack du style désiré, la prochaine étape consiste à la désarchiver, à moins de télécharger directement le pack non archivé, dans le répertoire des styles, styles/. Ensuite, allez dans Administration -> Styles, vous devriez voir listé votre nouveau style, cliquez sur installer et il sera disponible pour vos utilisateurs.

Attention : si vous créez votre propre style, ou si vous modifiez un style existant, surtout n'oubliez pas de mettre à "oui" la propriété "Recompiler les différents éléments du style:", dans l'écran Admin->Général->Paramètres de charge. Ce paramètre permet au système de cache de détecter des modifications apportées aux styles, et de les prendre en compte automatiquement. Si ce paramètre est désactivé, vous ne verrez jamais vos modifications être prises en compte.

2.iii. Les MODifications, ou "MODs"

Bien qu'officiellement non soutenue par le groupe phpBB, il existe une offre florissante de MODifications, ou "MODs", pour phpBB. Ces MODs, élaborées par des tiers, étendent les fonctionnalités de la version standard de phpBB, elles peuvent être consultées à l'adresse suivante :

http://www.phpbb.com/mods/
voire la base de données des MODs de phpBB-fr.com

Attention : tout bogue ou toute conséquence de l'ajout d'un MOD à votre installation phpBB NE DOIT PAS être reportée dans le "bug tracker" (voir ci-dessous). Commencez déjà par retirer le MOD et par vérifier que le problème persiste.

Rappelez-vous également que toute modification qui impacte la base de données, de quelque manière que ce soit, peut rendre considérablement difficile toute mise à jour ultérieure de votre forum vers les futures versions. Ceci étant dit, de nombreux utilisateurs ont utilisés, et continuent d'utiliser, un grand nombre de MODs disponibles, et ce avec un réel succès.

3. Obtenir de l’aide sur phpBB3

phpBB3 peut sembler un peu intimidant pour les nouveaux utilisateurs, notamment son système de permissions. La première chose à faire consiste à consulter la FAQ [ndt : FAQ (vo)], qui couvre des questions fondamentales pour commencer. Si vous avez besoin d'aide supplémentaire, voici plusieurs sources que vous pouvez consulter.

3.i. La documentation phpBB3

Une documentation complète est désormais disponible en ligne, et peut être consultée à l'url suivant :

http://www.phpbb.com/support/documentation/3.0/

Cette documentation couvre toutes les fonctionnalités, depuis les aspects liés à l'installation jusqu'à ceux liés à la gestion des utilisateurs et au paramétrage de leurs permissions.

3.ii. Les forums de la communauté

Le groupe phpBB entretient une communauté prospère, dans laquelle un certain nombre de personnes ont généreusement décidé de faire don de leur temps, et de leur connaissance, pour assister les utilisateurs. Ce site peut être consulté à l'url suivante :

http://www.phpbb.com/

Si vous demandez de l'aide par le biais de ce forum, commencez par effectuer une recherche avant de poster toute demande de support. Cette démarche économisera du temps aussi bien à vous, qu'à nous ; et surtout, elle permettra aux groupes de développeurs, de modérateurs et d'assistance, de consacrer plus de temps à résoudre des problèmes ou analyser des questions encore sans réponse. Rappelez-vous aussi que phpBB est un projet qui s'appuie intégralement sur des bénévoles qui ne reçoivent aucune compensation pour le temps qu'ils donnent, et cela inclut bien sûr les modérateurs et les développeurs. Donc, soyez respectueux et attentionné en attendant une réponse à votre question.

3.iii Le tchat, ou "irc"

Une autre source pour pouvoir trouver de l'aide est notre canal "IRC", pour "Internet Relay Chat". Cela fonctionne sur le réseau d'IRC Freenode, irc.freenode.net, et le canal en est #phpbb et peut être consulté par tout bon client IRC tel que mIRC, XChat, etc. Encore une fois, n'abusez pas de ce service et restez respectueux des autres utilisateurs.

4. Statut de cette version

Ceci est la troisième version stable de phpBB. Les fonctionnalités de la gamme 3.0.x sont globalement figées, le dernier numéro permettant de suivre les versions corrigeant des bogues ou des questions de sécurité. Toutefois, des modifications de certaines fonctionnalités, voire l'ajout de certaines fonctionnalités mineures, peuvent être faites si cela est jugé absolument nécessaire. Notre prochaine version majeure de phpBB sera la 3.2, et l'étape de plannification a débuté (la version instable de développement est la 3.1). Toutefois, merci de ne pas poster de question sur la date de sortie de la 3.2, aucune date n'est actuellement définie.

Ceux qui s'intéresseraient au développement de phpBB peuvent garder un oeil sur les forums de la communauté afin de voir comment les choses progressent, à l'url :

http://area51.phpbb.com/phpBB/

Notez que ce forum NE DOIT PAS être utilisé pour obtenir de l'assistance, ou même poser des questions, sur phpBB 2.0.x ou 3.0.x. C'est le forum principal de la communauté qui est l'endroit approprié. Aussi, de tels posts seront verrouillés et laissés sans réponse.

5. Rapporter des bogues

Le groupe phpBB utilise un système de suivi des bogues, ou "bug tracker", pour enregistrer, recenser et gérer les bogues, système qui peut être trouvé à l'url indiquée ci-dessous. Merci de NE PAS POSTER de rapport de bogue dans nos forums, car la discussion sera verrouillée. En outre, merci de NE PAS UTILISER le "bug tracker" pour des demandes d'assistance. La publication d'une telle demande sera redirigée à chaque fois vers le forum d'aide, en consommant au passage du temps de travail qui aurait pu être consacré au véritable débogage.

http://www.phpbb.com/bugs/

Bien que nous apprécions de recevoir des rapports de bogue, plus nous en recevons plus phpBB sera stable, nous vous demandons de respecter les étapes suivantes pour signaler un nouveau bogue :

  • Tout d'abord, déterminez si votre bogue est reproductible. La manière de vérifier cela dépend du bogue et de sa nature. Seule la reproductibilité du bogue est le signe que le problème concerne phpBB3, ou s'y rapporte. Si un bogue n'est pas reproductible, cela peut provenir de votre hébergeur ayant mis en oeuvre quelque chose, d'un utilisateur ayant accompli quelque chose d'absurde, etc. Les rapports de bogue pour des événements non reproductibles peuvent nous ralentir dans la résolution de corriger de réels bogues reproductibles.

  • Ensuite, cherchez parmi les bogues déjà signalés pour voir si votre bogue, ou un bogue similaire au votre, n'est pas recensé. Si tel est le cas, merci alors de compléter ce bogue existant plutôt que de créer un nouveau bogue finallement identique, cela ne ferait que nous ralentir.

  • Puis vérifiez dans les forum, grâce au système de recherche, si quelqu'un n'a pas déjà abordé un sujet qui ressemble au votre. Toutefois, comme précisé plus haut, merci de NE PAS POSTER votre bogue, sauf s'il s'agit soit d'un bogue non reproductible, soit d'un bogue dont vous êtes sûr qu'il est la conséquence d'une action de votre part indépendamment de phpBB3.

  • Enfin, si votre bogue est reproductible et n'est pas recensé, alors merci de nous le signaler et de l'ajouter à la liste des bogues.

Si vous poster finalement un nouveau bogue, i.e. un bogue qui n'est pas encore listé dans le "bug tracker", assurez-vous tout d'abord de vous être préalablement connecté, sous votre pseudo et avec le mot de passe que vous utilisez sur les forums de la communauté, et ensuite, merci de fournir les informations suivantes :

  • Le type et la version de votre serveur HTTP, par exemple : Apache 1.3.28, IIS 4, Sambar, etc.
  • La version de PHP et son mode opératoire, par exemple : PHP 5.1.1 en tant que module, PHP 4.4.4 s'exécutant comme CGI, etc.
  • Le type et la version de votre système gérant votre base de données, par exemple : MySQL 4.0.1, PostgreSQL 7.3.2, MSSQL Serveur 2000 SP1, etc.

Les type et version de votre gestionnaire de base de données sont indiqués dans le panneau de contrôle d'administration

Merci également de fournir votre rapport de manière aussi détaillé que possible, en particulier lister les étapes qui permettent d'aboutir au problème. De plus, si vous disposez d'un correctif dont vous êtez sûr qu'il fonctionne, qui est conforme à nos recommandations de codage [ndt : coding guidelines (vo)] , et qui n'introduit pas d'autres problèmes ou d'incompatibilité, alors merci de nous le signaler. Cependant ne l'incluez que dans votre rapport sur le bogue, si vous le devez, en cas de besoin nous vous le demanderons.

Une fois un bogue soumis, un suivi des commentaires ajoutés sera mis en place pour vous prévenir par messagerie. Merci de fournir toute information complémentaire qui pourrait vous être demandée ! En effet, il est frustrant pour nous de recevoir un rapport de bogue, de demander un complément d'information, sans rien obtenir de plus. Dans ce type de situation, nous appliquons donc un protocole qui clôture le bogue, même si nous risquons de ne pas prendre en compte un problème réel. Bien entendu, nous préférerions ne pas avoir à en arriver à cette situation.

5.i. Les bogues de sécurité

Si vous trouvez une vulnérabilité potentiellement lié à la sécurité de phpBB, surtout NE PAS poster cette information dans le "bug tracker", dans des forums plubliques, etc. ! Le faire permettrait en effet à des personnes peu scrupuleuses de tirer avantage de cette vulnérabilité, avant que nous ayions le temps de mettre au point un correctif. Tout bogue lié à la sécurité devrait être enregistré dans notre système de suivi de bogue de sécurité, "security tracker" :

http://www.phpbb.com/security/

6. Résumé de la liste des bogues en cours

Cette liste, bien que non exhaustive, présente les bogues qui peuvent concerner les utilisateurs sur une large échelle. D'autres bogues, repertoriés dans le système de suivi des bogues, s'avèrent limités à certaines configurations, certains types d'installation, certaines mises à jour ou conversions.

  • Le changement de version peut ne pas se terminer pour de grands forums sur certains systèmes ;
  • La mise à jour d'une version peut échouer sur de grosses mises à jour sur certains systèmes ;
  • Un émoticone [ndt : smiley] placé juste derrière un bbcode ne sera pas analysé. Un émoticone doit toujours être encadré d'espaces.

7. Problèmes de compatibilité avec PHP

phpBB ne prend plus en charge PHP3 du fait de divers problèmes de compatibilité, aussi nous vous conseillons de mettre à jour votre version de PHP pour la dernière version stable, à savoir PHP5, pour faire fonctionner phpBB. La version minimale requise est PHP 4.3.3.

Rappelez-vous que faire fonctionner n'importe quelle application sur une version en développement de PHP peut avoir un résultat étrange et inattendu, qui peut sembler être un bogue de l'application, sans que cela soit le cas. Par conséquent, nous vous recommandons de mettre à jour votre version de PHP pour la dernière version stable de PHP avant de faire fonctionner phpBB3. Au cas où vous utiliseriez une version de développement de PHP, merci de vérifier que votre bogue se produit aussi avec une version stable de PHP, avant de l'enregistré dans le "bug tracker".

Ce forum a été développé et testé sur les système Linux et Windows, entre autres, en utilisant un service Apache et les système de gestion de bases de données MySQL 3.23, 4.x, 5.x, MSSQL Server 2000, PostgreSQL 7.x, Oracle 8, SQLite et Firebird. Les versions de PHP testées vont de 4.3.3 à 6.0.0-dev, et sans rencontré de problème.

7.i. Avis de sécurité sur PHP

Actuellement aucun problème de sécurité n'est connu concernant PHP.
